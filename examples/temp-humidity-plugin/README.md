# weather-plugin

An EOS IoT plugin written in Rust that fetches real weather data from
api.met.no, then connects to the EOS agent via Phoenix Channels and publishes
NGSI-LD WeatherObserved entities.

## How it works

```
Sensor (Rust) ──WS──► EOS Agent (Elixir) ──HTTP──► NGSI-LD Broker (Phaeton)
```

1. Connects to `IOT_AGENT_WS_URL/websocket?token=PLUGIN_TOKEN`
2. Joins Phoenix channel `plugin:{PLUGIN_ID}`
3. Sends `register` with `entity_type_uri = "WeatherObserved"`
4. Creates an initial NGSI-LD entity (`entity_create`) with city-based URN
5. Includes GeoJSON location in the entity payload for map rendering
6. Every 10 min: publishes MET temperature and humidity updates (`entity_update`)
6. Every 30 s: sends a Phoenix heartbeat so the channel stays alive

The plugin never speaks directly to the broker — all NGSI-LD HTTP calls
are handled by the EOS agent.

## Environment variables

| Variable          | Description                                          |
|-------------------|------------------------------------------------------|
| `PLUGIN_ID`       | Unique plugin name (e.g. `temp-sensor-1`)            |
| `PLUGIN_TOKEN`    | HMAC auth token generated by the EOS operator        |
| `IOT_AGENT_WS_URL`| Agent WebSocket base URL (e.g. `ws://eos-agent/plugins`) |
| `CITY_NAME`       | City used for URN suffix (default `Braunschweig`)    |
| `CITY_LAT`        | City latitude (default `52.2689`)                    |
| `CITY_LON`        | City longitude (default `10.5268`)                   |
| `RUST_LOG`        | Log filter (default `plugin=info`)                   |

All three required vars are injected automatically when the plugin is
deployed via a `Plugin` CRD.

## Running locally (development)

Start the EOS agent first, then run the plugin with the correct env vars:

```bash
export PLUGIN_ID=temp-sensor-dev
export PLUGIN_TOKEN="$(mix run --no-start -e \
  'IO.puts Eos.Plugins.Token.generate("weather-plugin")')"
export IOT_AGENT_WS_URL=ws://localhost:4000/plugins
export CITY_NAME=Braunschweig
export CITY_LAT=52.2689
export CITY_LON=10.5268

cargo run
```

## Building the Docker image

```bash
docker build -t weather-plugin:latest .
```

## Deploying with the EOS operator

1. Create a `Broker` CR pointing at your NGSI-LD broker (see `priv/k8s/crds/broker.yaml`).
2. Apply the example `Plugin` CR:

```bash
kubectl apply -f plugin.yaml
```

The operator will:
- Generate a token secret (`eos-plugin-weather-plugin-token`)
- Start a pod with the correct env vars injected
- Manage the plugin lifecycle

See `plugin.yaml` for the full example manifest.
